# План: кнопка «Отправить в таблицу» и выгрузка в Google Sheets

## Рекомендация: Google Sheets

**Почему Sheets, а не только БД:**
- Уже есть Postgres (таблица `extractions`), но смотреть данные удобнее в таблице с колонками.
- **Google Sheets** — бесплатно, привычный интерфейс, можно делить доступ, строить графики, фильтры.
- Альтернативы: Airtable (удобно, но лимиты на бесплатном плане), Notion (API сложнее). Для «столбцы = параметры идеи» Sheets оптимален.

**Связка в итоге:** Telegram-бот → при нажатии кнопки → добавление строки в Google-таблицу (через Google Sheets API и сервисный аккаунт).

---

## Что нужно сделать по шагам

### 1. Парсинг ответа бота в структуру (поля для таблицы)

Сейчас бот присылает текст вида:
```
Название 1: ...
Название 2: ...
Название 3: ...
Автор идеи: ...
Аналитик(и): ...
...
Источник: ...
```

Нужно один раз описать парсер: по строкам «Ключ: значение» и блоку «Драйверы роста:» собрать объект с полями:
- Название 1, Название 2, Название 3
- Автор идеи, Аналитик(и), Базовый актив, Целевая цена, Стоп-лосс, Направление, Горизонт (дней)
- Драйвер 1, Драйвер 2, Драйвер 3, Драйвер 4, Драйвер 5 (парсить из блока «Драйверы роста» по строкам; если меньше пяти — остальные пустые)
- Ссылка (значение из строки «Источник»)
- Дата добавления

Этот объект будет использоваться и для сохранения в БД, и для строки в Sheets.

### 2. Кнопка под сообщением бота

- В Telegram под сообщением с идеей показывать **Inline-кнопку**: «Отправить в таблицу».
- Для этого при отправке ответа вызывать не просто `sendMessage`, а `sendMessage` с `reply_markup`: одна кнопка с `callback_data`, например `send_sheet`.
- Ограничение: `callback_data` до 64 байт, поэтому туда нельзя вложить все данные — только метку. Идентификатор записи берём из БД.

### 3. Связь «нажатие кнопки» ↔ запись в БД

- При ответе пользователю бот уже сохраняет данные в Postgres (`saveExtraction`). Нужно дополнительно сохранять **id сообщения бота** (message_id ответа), чтобы по нажатию кнопки понимать, к какой идее оно относится.
- Вариант А: добавить в таблицу `extractions` колонку `bot_message_id`, при сохранении передавать в неё `message_id` из ответа Telegram после `sendMessage`.
- Вариант Б: не использовать БД — тогда при нажатии кнопки нечем однозначно найти данные (в callback_data мало места). Поэтому **нужна БД** (Postgres уже есть).

Итого: после отправки ответа с кнопкой получаем из API Telegram `message_id` этого сообщения, сохраняем его в `extractions` (или в отдельной таблице связки). При нажатии кнопки в обработчике `callback_query` имеем `chat_id` и `message.id` (сообщение с кнопкой) → по ним находим запись в БД → по ней получаем распарсенные поля → отправляем одну строку в Sheets.

### 4. Обработка нажатия кнопки (callback_query)

- В webhook кроме `message` приходит и `callback_query` (нажатие inline-кнопки).
- Нужно в `index.ts` (или отдельном обработчике): если в update есть `callback_query`, то:
  - взять `callback_query.message.chat.id` и `callback_query.message.message_id`;
  - по ним найти запись в `extractions` (по `chat_id` и `bot_message_id`);
  - из записи взять `raw_output` (или уже сохранённые структурированные поля) → распарсить, если ещё не парсили при сохранении;
  - вызвать функцию «добавить строку в Google Sheets»;
  - ответить на callback: `answerCallbackQuery` (убрать «часики») и при желании показать уведомление «Добавлено в таблицу».

### 5. Интеграция с Google Sheets

- **Google Cloud:** создать проект → включить **Google Sheets API**.
- **Сервисный аккаунт:** создать сервисный аккаунт, скачать JSON с ключом. Этот email (типа `...@...iam.gserviceaccount.com`) дать доступ к таблице: «Поделиться» → редактор (или только запись в ячейки).
- **Таблица:** первая строка — заголовки: Название 1, Название 2, Название 3, Автор идеи, Аналитик(и), Базовый актив, Целевая цена, Стоп-лосс, Направление, Горизонт (дней), Драйвер 1 … Драйвер 5, Ссылка, Дата. Порядок столбцов может быть любым — в коде подстановка по названию заголовка.
- В коде: библиотека `googleapis` (пакет `googleapis` в npm). Инициализация через сервисный аккаунт, метод `spreadsheets.values.append` — добавить одну строку (массив значений по колонкам) в конец листа.
- Переменные окружения (Railway): путь к JSON ключу или сам JSON (например `GOOGLE_SHEETS_CREDENTIALS_JSON`) и `GOOGLE_SHEET_ID` (id таблицы из URL).

### 6. Если БД не используется (только Sheets)

Если решим не использовать Postgres для этого сценария:
- Тогда в `callback_data` можно положить только короткий идентификатор. Вариант: сохранять последние N ответов в памяти (Map по `chat_id + message_id` бота) и по ним отдавать данные в Sheets. Минус: при перезапуске бота данные теряются. Поэтому **предпочтительно опираться на Postgres** и хранить там и `raw_output`, и при желании уже распарсенный JSON полей.

## Порядок внедрения (с чего начать)

1. **Парсер текста ответа** — функция `parseIdeyaBlock(replyText: string) → Record<string, string>`, которая по текущему формату (Название 1:, Автор идеи:, …) возвращает объект полей. Покрыть тестами или вручную проверить на реальном ответе.
2. **БД:** добавить колонку `bot_message_id` в `extractions`, изменить `saveExtraction` так, чтобы принимала и сохраняла `botMessageId`; при сохранении вызывать после `sendMessage` и передавать полученный от Telegram `message_id`.
3. **Telegram:** расширить `sendMessage` (или добавить `sendMessageWithButton`) — опционально передавать `reply_markup` с одной Inline-кнопкой «Отправить в таблицу» с `callback_data: "send_sheet"`. После отправки возвращать из функции `message_id` отправленного сообщения.
4. **Webhook:** различать в update `message` и `callback_query`. Для `callback_query` — извлечь `chat_id`, `message.message_id`, по ним найти запись в БД, распарсить `raw_output` через `parseIdeyaBlock`, вызвать добавление строки в Sheets, ответить на callback.
5. **Google Sheets:** модуль (например `src/sheets.ts`) — инициализация из env, функция `appendIdeyaRow(fields: Record<string, string>)`. В Railway задать переменные и положить JSON ключа сервисного аккаунта.
6. **Заголовки и колонки:** один раз зафиксировать соответствие «ключ в объекте → заголовок колонки» и порядок колонок в таблице (например, массив полей в нужном порядке для `append`).

После этого сценарий «нажал кнопку → одна строка в таблице с полями идеи» будет работать. Дальше можно доработать: несколько листов, фильтры, обновление существующей строки и т.д.
